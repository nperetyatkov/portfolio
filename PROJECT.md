# PROJECT — Portfolio

## 1. Цель
Персональный сайт-портфолио на чистом HTML/CSS/JavaScript с простой, понятной и масштабируемой структурой.

## 2. Текущий статус
- Рабочая ветка: `new-structure`
- Legacy-версия зафиксирована в `main` и не участвует в текущей разработке
- Главная страница сверстана: `header`, `hero`, `about`, `projects`, `contact`, `footer`
- Добавлена и подключена страница кейса: `pages/project-mosru.html` (единый шаблон под проекты)
- Подключён интерактив:
  - мобильное меню + активное состояние ссылок в мобильной навигации;
  - плавный скролл к якорям (с учётом фиксированного хедера);
  - desktop dropdown-навигация по секциям кейса (со слежением за активной секцией);
  - единый источник данных пунктов меню кейса через `case-nav-data` (desktop + mobile рендерятся из одного списка);
  - кастомный курсор (desktop) с безопасным включением: скрытие системного курсора только при активном `.cursor`;
  - `hover`-эффекты и кастомный курсор ограничены desktop/tablet-режимом (`min-width: 800px` + `hover: hover` + `pointer: fine`), на мобильном режиме они отключены;
  - копирование телефона по клику на desktop/tablet;
  - авторасстановка неразрывных пробелов в тексте;
  - просмотр изображений кейса в lightbox (клик/Enter/Space, закрытие по `Esc`);
  - для изображений кейса добавлен `press`-эффект и плавная анимация открытия/закрытия lightbox (fade + scale/translate).

## 3. Технологии
- HTML5
- CSS (слои: `base`, `layout`, `components`, `pages`)
- Vanilla JavaScript (ES modules)
- Git + GitHub

## 5. Архитектурные правила
- Один файл = одна зона ответственности.
- Новые файлы добавляются только при реальной потребности (переиспользование, независимая логика, перегрузка текущего файла).
- Слои CSS не смешиваются:
  - `base` — токены, типографика, глобальная база
  - `layout` — каркас и крупные блоки
  - `components` — переиспользуемые элементы
  - `pages` — стили конкретной страницы
- Инициализация интерактива — через `scripts/main.js`.
- Компоненты не должны зависеть друг от друга напрямую.

## 6. Правила изменений
- Один логический шаг = один коммит.
- После каждого коммита сайт должен оставаться рабочим.
- Рефакторинг и новая функциональность не смешиваются в одном коммите.
- Любые заметные структурные изменения сначала коротко фиксируются в этом файле.
- После каждого крупного/значимого изменения обязательно обновлять `PROJECT.md` (минимум: раздел «Текущий статус» и при необходимости Code Map/правила).
- Для веса шрифта использовать только токены (`--fw-regular`, `--fw-medium`, `--fw-semibold`, `--fw-bold`); не использовать `font-variation-settings` и ручные значения вида `"wght" ...`.

## 7. Ориентир по дизайну
Основной ориентир: https://nperetyatkov.figma.site

Принципы:
- минималистичная подача;
- акцент на типографику, отступы, иерархию;
- аккуратные анимации без перегрузки интерфейса.

## 8. Формат работы
- Сначала уточняем цель задачи и затрагиваемые файлы.
- Дальше идём небольшими проверяемыми шагами.
- При риске регрессии сначала согласуем подход, потом правим код.
